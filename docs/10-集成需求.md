# 10 集成需求（Integration）

> 对应 MPDV 产品规格书模块：MIP-ECS / P_ECS-ERP / P_ECS-CLD（企业连接服务）与 P_DEC-BF / IOT-BC1（分布式边缘采集）。业务系统集成由 `xmes-core 的 integration 模块`（≈ECS）承担；设备采集（≈DEC）由**后期独立物联网模块**承担，MES 侧只定义对接契约（见 01 §2 映射表、决策 Q6）。

## 1. ERP 集成（FR-INT-1x）

| 编号 | 需求 | 方向 | 优先级 |
| --- | --- | --- | --- |
| FR-INT-11 | 主数据同步：物料、BOM、工艺路线（可配置主数据归属：ERP 主 / MES 主）、供应商、客户 | ERP→MES | P0 |
| FR-INT-12 | 工单同步（对应 P_ECS-ERP）：工单下达/变更/取消指令接收，含工序、并行/替代工序序列、组件清单、工装文档链接、长文本、订单网络与生产变体、自定义字段；状态回执 | ERP→MES | P0 |
| FR-INT-13 | 生产回传：作业**结果记录**（完工数量、报废、实际工时、各时间账户时长）、物耗（倒冲清单）按事件/按班次批量回传；含修正产生的**冲销结果记录** | MES→ERP | P0 |
| FR-INT-14 | 库存协同：收货批次接收；完工入库交接与回执；库存移动流水上传（对 WMS 同理） | 双向 | P0 |
| FR-INT-15 | 集成方式（对应 MIP-ECS）：可插拔适配器体系——REST/SOAP、**SAP RFC**、文件（XML/JSON/CSV/定长）、中间表/直连 DB（老 ERP）、云 ERP 连接器（请求-转换-上传模式）；**传输表机制**：入站数据先落传输表再经转换入对象模型，出站同理；**分发模型**配置各逻辑系统订阅的数据范围；按客户 ERP（用友/金蝶/SAP）做适配层 | — | P0 |
| FR-INT-16 | 集成监控：接口日志、失败重试（指数退避）、死信人工补办界面、对账报表（MES vs ERP 完工数） | — | P0 |

## 2. 设备接入（FR-INT-2x）

> 决策 Q6：FR-INT-21~25 整体划归**后期独立物联网模块**项目（对应 DEC 能力），不在 MES 首期范围。MES 本期交付的是双方的**对接契约**（FR-INT-20）。工位外设（扫码枪/打印机）不属于设备采集，仍在 MES 范围（FR-INT-26）。

| 编号 | 需求 | 优先级 |
| --- | --- | --- |
| FR-INT-20 | **设备事件接入 API（MES 侧契约）**：物联网模块向 MES 推送设备事件的标准接口——事件信封（设备编码、事件类型：状态/报警/计数/过程值、值、源时间戳、序号），鉴权（OAuth2 客户端凭证）、幂等去重、乱序与补传容忍；MES 侧消费链路（状态记录、自动报工挂点、质量事件）按此契约预留 | P0 |
| FR-INT-21 | 协议支持（对应 DEC）：OPC UA（首选）、MQTT、Modbus TCP；预留 **EUROMAP 77**（注塑机）、**MTConnect**（机床）、S7、FOCAS（FANUC）、串口适配插件位 | 后期·物联网 |
| FR-INT-22 | 采集配置：DataTag 配置（02 §3.2）热更新到采集器；**逻辑模块**信号预处理与虚拟采集点；计数值/状态值/数字 IO 采集；基于节拍/时钟/运行信号的产线状态自动监控；过程值采集与公差监控；目标值从控制器读取 | 后期·物联网 |
| FR-INT-23 | 边缘缓存与分发：断网 store-and-forward，恢复后按时间序补传；采集数据经 **MQTT** 分发供在线可视化订阅；时序存储 | 后期·物联网 |
| FR-INT-24 | 反向控制：NC 程序/设定数据下载与回传（DNC）、配方/参数下发、**生产锁**启停；写操作需白名单与审计 | 后期·物联网 |
| FR-INT-25 | 接入健康度：通道/设备在线状态、采集频率、丢点率监控页面 | 后期·物联网 |
| FR-INT-26 | 工位外设接入规范（MES 范围）：扫码枪（键盘仿真）、标签打印机（CPCL/ZPL 网络直打）、电子秤/卡尺（串口直读） | P0（扫码/打印），P2（量具/秤） |

## 3. 开放 API 与消息（FR-INT-3x）

| 编号 | 需求 | 优先级 |
| --- | --- | --- |
| FR-INT-31 | OpenAPI：核心对象（工单/作业/报工/批次/检验）REST API，OAuth2 客户端凭证授权，按客户端限流；自动生成 OpenAPI 3 文档 | P1 |
| FR-INT-32 | Webhook/事件订阅：第三方订阅生产事件（签名校验、重试） | P2 |
| FR-INT-33 | 消息推送通道：企业微信/钉钉机器人 Webhook 用于升级与报表推送 | P1 |

## 4. 标准参照

- 与 ERP 的业务对象交换语义参照 ISA-95 Part 2 / B2MML（不强制报文格式，但字段语义对齐，方便未来标准化）。
- 设备信息模型优先采用 OPC UA 行业伴随规范（如 umati）映射到 DataTag。
