# 10 集成需求（Integration）

> 对应 MPDV 产品规格书模块：MIP-ECS / P_ECS-ERP / P_ECS-CLD（企业连接服务）与 P_DEC-BF / IOT-BC1（分布式边缘采集）。xMes 以 `xmes-core 的 integration 模块`（业务系统集成，≈ECS）与 `xmes-connect`（设备接入，≈DEC）承担（见 01 §2 映射表）。

## 1. ERP 集成（FR-INT-1x）

| 编号 | 需求 | 方向 | 优先级 |
| --- | --- | --- | --- |
| FR-INT-11 | 主数据同步：物料、BOM、工艺路线（可配置主数据归属：ERP 主 / MES 主）、供应商、客户 | ERP→MES | P0 |
| FR-INT-12 | 工单同步（对应 P_ECS-ERP）：工单下达/变更/取消指令接收，含工序、并行/替代工序序列、组件清单、工装文档链接、长文本、订单网络与生产变体、自定义字段；状态回执 | ERP→MES | P0 |
| FR-INT-13 | 生产回传：作业**结果记录**（完工数量、报废、实际工时、各时间账户时长）、物耗（倒冲清单）按事件/按班次批量回传；含修正产生的**冲销结果记录** | MES→ERP | P0 |
| FR-INT-14 | 库存协同：收货批次接收；完工入库交接与回执；库存移动流水上传（对 WMS 同理） | 双向 | P0 |
| FR-INT-15 | 集成方式（对应 MIP-ECS）：可插拔适配器体系——REST/SOAP、**SAP RFC**、文件（XML/JSON/CSV/定长）、中间表/直连 DB（老 ERP）、云 ERP 连接器（请求-转换-上传模式）；**传输表机制**：入站数据先落传输表再经转换入对象模型，出站同理；**分发模型**配置各逻辑系统订阅的数据范围；按客户 ERP（用友/金蝶/SAP）做适配层 | — | P0 |
| FR-INT-16 | 集成监控：接口日志、失败重试（指数退避）、死信人工补办界面、对账报表（MES vs ERP 完工数） | — | P0 |

## 2. 设备接入（FR-INT-2x，xmes-connect）

| 编号 | 需求 | 优先级 |
| --- | --- | --- |
| FR-INT-21 | 协议支持（对应 DEC）：OPC UA（首选）、MQTT、Modbus TCP；预留 **EUROMAP 77**（注塑机）、**MTConnect**（机床）、S7、FOCAS（FANUC）、串口适配插件位 | P0（OPC UA/MQTT），P1（Modbus），P2（其余） |
| FR-INT-22 | 采集配置下发：DataTag 配置（02 §3.2）热更新到采集器，无需重启；**逻辑模块**信号预处理与虚拟采集点（运算/组合生成）；计数值（良/废）、状态值、数字 IO 采集；基于节拍/时钟/运行信号的产线状态自动监控；过程值采集支持自动/周期/事件触发并做公差监控；目标值从控制器读取 | P0 |
| FR-INT-23 | 边缘缓存与分发：断网 store-and-forward，恢复后按时间序补传；事件带源时间戳；采集数据经 **MQTT** 分发供在线可视化订阅；采集实例运行状态可视 | P1 |
| FR-INT-24 | 反向控制：NC 程序/设定数据经边缘组件下载到机床控制器与上传回传（DNC）、配方/参数下发；**生产锁**启停；写操作需白名单与审计 | P2 |
| FR-INT-25 | 接入健康度：通道/设备在线状态、采集频率、丢点率监控页面 | P1 |
| FR-INT-26 | 外设接入规范：扫码枪（键盘仿真）、电子秤/卡尺（串口，对应 IOT-BC1 connect a scale）、标签打印机（CPCL/ZPL 网络直打） | P0（扫码/打印），P2（量具/秤） |

## 3. 开放 API 与消息（FR-INT-3x）

| 编号 | 需求 | 优先级 |
| --- | --- | --- |
| FR-INT-31 | OpenAPI：核心对象（工单/作业/报工/批次/检验）REST API，OAuth2 客户端凭证授权，按客户端限流；自动生成 OpenAPI 3 文档 | P1 |
| FR-INT-32 | Webhook/事件订阅：第三方订阅生产事件（签名校验、重试） | P2 |
| FR-INT-33 | 消息推送通道：企业微信/钉钉机器人 Webhook 用于升级与报表推送 | P1 |

## 4. 标准参照

- 与 ERP 的业务对象交换语义参照 ISA-95 Part 2 / B2MML（不强制报文格式，但字段语义对齐，方便未来标准化）。
- 设备信息模型优先采用 OPC UA 行业伴随规范（如 umati）映射到 DataTag。
