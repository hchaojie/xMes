# 08 功能需求 · 装配与现场执行（Assembly & Shopfloor Execution）

> 对应 MPDV HYDRA X "Assembly Management" 类 mApps（复杂装配流程的工位引导、操作员指引）以及 HYDRA 的 AIP（采集终端）形态。本模块定义**工位终端应用**的统一需求——它是操作工与 MES 交互的唯一界面，订单/物料/质量/设备各模块的现场功能都通过它落地。

## 1. 工位终端框架（FR-SHF-1x）

| 编号 | 需求 | 优先级 |
| --- | --- | --- |
| FR-SHF-11 | 终端注册与绑定：终端设备（触屏一体机/平板）绑定 Workplace，开机自动进入该工位上下文；一终端可服务多工位（切换） | P0 |
| FR-SHF-12 | 上岗会话：刷卡/扫码/账号三种上岗方式；多人同时上岗；无操作自动锁屏 | P0 |
| FR-SHF-13 | UI 规范：全屏、≥16pt 字号、大按钮（戴手套可点）、深色主题可选、所有关键操作可纯扫码完成（扫码枪键盘仿真） | P0 |
| FR-SHF-14 | 功能卡片化：终端首页按工位类型配置功能卡片（任务/报工/上料/自检/点检/报修/安灯/交接班），不同工位类型不同布局 | P0 |
| FR-SHF-15 | 弱网容错：网络中断时报工操作本地排队，恢复后按序补传；终端显示离线标识 | P1 |
| FR-SHF-16 | 消息与待办：派工变更、检验通知、安灯响应等实时推送到相关终端 | P0 |

## 2. 电子作业指导（FR-SHF-2x）

| 编号 | 需求 | 优先级 |
| --- | --- | --- |
| FR-SHF-21 | SOP 展示：开工后自动展示当前工序的作业指导（PDF/图片/视频），与工艺版本一致（来自工艺快照） | P0 |
| FR-SHF-22 | 步骤化作业引导：复杂装配工序拆为步骤清单，逐步确认（勾选/扫码/拍照），支持步骤级防错条件 | P1 |
| FR-SHF-23 | 工程变更提示：SOP 新版本生效后首次打开强制阅读确认（留痕） | P1 |
| FR-SHF-24 | 参数面板：当前工序工艺参数目标值/上下限展示；需人工记录的参数在此录入 | P1 |

## 3. 防错（Poka-Yoke）（FR-SHF-3x）

统一的防错校验框架，校验项按工序配置启停（部分已在各模块定义，此处汇总为框架需求）：

| 编号 | 需求 | 优先级 |
| --- | --- | --- |
| FR-SHF-31 | 防错规则配置：按工序勾选启用——前道完成、质量门放行、首件通过、资质、批次状态、BOM 物料匹配、FIFO/FEFO、模具匹配、NC 程序版本、参数在限；每项可设拦截/警告 | P0 |
| FR-SHF-32 | 顺序防错：序列件按工艺路线顺序流转，跳站扫码即拦截 | P1 |
| FR-SHF-33 | 外设联动防错：拧紧枪/螺丝机结果采集，达不到目标扭矩不允许过站（P2，按行业需要） | P2 |
| FR-SHF-34 | 防错放行：拦截可由班组长授权放行（刷卡二次认证），全程留痕 | P0 |

## 4. 安灯（Andon）（FR-SHF-4x）

| 编号 | 需求 | 优先级 |
| --- | --- | --- |
| FR-SHF-41 | 安灯呼叫：终端一键呼叫（缺料/质量/设备/工艺求助），状态流：呼叫→响应→解决→关闭，各节点时间留痕 | P0 |
| FR-SHF-42 | 安灯分发与升级：按类型路由到责任角色（推送 PDA/看板/消息）；超时未响应按升级链上报（接 09 升级管理） | P0 |
| FR-SHF-43 | 安灯看板：车间大屏实时显示呼叫中工位与时长；安灯统计（响应时长、Top 原因） | P1 |

## 5. 交接班（FR-SHF-5x）

| 编号 | 需求 | 优先级 |
| --- | --- | --- |

| FR-SHF-52 | 班会看板：班前会一页式视图（昨日产量/质量/今日计划/异常） | P2 |
