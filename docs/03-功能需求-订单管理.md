# 03 功能需求 · 订单管理（Order Management）

> 对应 MPDV HYDRA X "Order Management" 类 mApps：Work Order Management（工单管理）、Detailed Scheduling/Leitstand（电子排程看板，经典 HLS）、Work Order Execution / Order Data Collection（工单执行与报工，经典 BDE）、Work Order Controlling（工单管控）。

涉及对象：`ProductionOrder`、`OperationTask`、`Booking`、`DispatchList`、`TimeTicket`（见 02 §5）。

## 1. 工单管理（FR-ORD-1x）

| 编号 | 需求 | 优先级 |
| --- | --- | --- |
| FR-ORD-11 | 工单接收：从 ERP 接口接收工单（物料、数量、计划起止、BOM/工艺引用），自动生成工艺快照与作业列表；接口异常进入待处理队列人工补办 | P0 |
| FR-ORD-12 | 手工建单：支持 MES 内手工创建工单（选物料→自动带出当前生效工艺路线与 BOM），用于无 ERP 场景与试制 | P0 |
| FR-ORD-13 | 工单下达：单个/批量下达；下达校验（工艺快照完整、关键物料可用性提示、所需资质/工装存在性提示），校验级别可配置为拦截或警告 | P0 |
| FR-ORD-14 | 工单变更：下达后允许改数量（不小于已报良品数）、改计划日期、改优先级；变更留痕；工艺变更须先撤回未开工作业 | P0 |
| FR-ORD-15 | 工单拆分/合并：按数量拆分子工单（追溯关联保留）；同物料同工艺工单合并排产（P2） | P1 |
| FR-ORD-16 | 工单挂起/恢复/取消/短关：状态机见 02 §5.2；短关（欠交关闭）需审批 | P0 |
| FR-ORD-17 | 重工工单：从不合格品处置发起，自动关联原工单与原批次，工艺路线选用返修路线 | P1 |
| FR-ORD-18 | 工单列表与详情：多维查询（状态/物料/产线/日期/延期标记）；详情页含工序进度甘特、报工记录、用料、检验、异常时间线 | P0 |

## 2. 作业排程 / 电子排程看板（FR-ORD-2x）

定位：首期提供**人工交互式甘特排程 + 规则自动排序**（对应 HYDRA Leitstand 的核心能力），自动优化算法（APS）列入 P2。

| 编号 | 需求 | 优先级 |
| --- | --- | --- |
| FR-ORD-21 | 排程甘特图：横轴时间（含班次/非工作时间底色）、纵轴 WorkCenter/Workplace 两级；作业条显示工单号/物料/数量/状态颜色；支持缩放（时/班/日/周）、虚拟滚动（≥200 工位、≥5000 作业流畅） | P0 |
| FR-ORD-22 | 拖拽排程：作业条拖拽改时间/改机台；拖拽时实时校验——设备能力匹配（工序默认工作中心/替代组）、前后工序顺序约束、工装/资质冲突提示；冲突以红色标记不阻断（计划员决策优先） | P0 |
| FR-ORD-23 | 规则自动排序：对选中范围按规则一键排序——交期优先(EDD)、先到先做(FIFO)、优先级、最短加工时间(SPT)、换型优化（同物料/同模具相邻），规则可组合加权 | P1 |
| FR-ORD-24 | 有限/无限产能模式：无限模式仅按工艺时间铺排并显示超载热力；有限模式禁止同机台时间重叠 | P0 |
| FR-ORD-25 | 作业拆分：在甘特上把一个作业按数量拆到多机台并行 | P1 |
| FR-ORD-26 | 排程基线与发布：排程结果先保存为草稿，发布后生成各工位派工队列（DispatchList）并推送终端；保留基线对比（计划 vs 实际偏差） | P0 |
| FR-ORD-27 | 实际进度回填：甘特上叠加显示实际开完工进度条；延期作业自动标红并可一键顺延后续 | P0 |
| FR-ORD-28 | 排程看板辅助信息：机台当日负荷率、缺料/模具不可用等约束角标、交期风险列表 | P1 |
| FR-ORD-29 | APS 自动优化排程（遗传/约束求解，考虑模具数量、人员技能等多资源约束） | P2 |

## 3. 生产报工 / 工单执行（FR-ORD-3x）

报工入口：工位终端（主）、PDA、管理端补录、设备自动报工（见 04 资源管理 FR-RES-2x）。

| 编号 | 需求 | 优先级 |
| --- | --- | --- |
| FR-ORD-31 | 上岗/开工：人员在终端刷卡/扫码上岗→从派工队列选作业（或扫工单流转卡）→开工；开工校验：前道完成度、质量门放行、资质、批次状态、NC 程序版本（防错项可按工序配置启停） | P0 |
| FR-ORD-32 | 报数：录入良品数、废品数（必选缺陷/原因码）、返工数；支持过程多次报数与完工一次性报数两种模式（按工序配置）；数量守恒校验（R5-1） | P0 |
| FR-ORD-33 | 暂停/恢复：暂停必选原因码（设备/缺料/换班/质检等待…），原因码联动设备状态 | P0 |
| FR-ORD-34 | 完工：触发后道可开工、质量门检验单生成、转移批量逻辑、模具模次累计、产出批次生成/入线边库 | P0 |
| FR-ORD-35 | 多人协同与多机操作：一作业多人上岗（工时按规则分摊：均摊/按角色系数）；一人看多机（巡回报工） | P1 |
| FR-ORD-36 | 报工冲销与补录：错报负向冲销（留痕）；漏报由班组长在管理端补录，超 24h 需审批；所有修正不改写原事件 | P0 |
| FR-ORD-37 | 外协作业：外协工序生成外协单，发出/接回登记数量与批次，接回触发 IQC | P1 |
| FR-ORD-38 | 异常报工待办：系统自动检出"开工未完工超时、数量不守恒、跨班未交接"等异常生成待办，闭环处理 | P1 |
| FR-ORD-39 | 工时票：由报工事件自动生成人员×作业工时票，支持计件单价挂接（输出给 ERP/HR，不在 MES 算薪） | P1 |

## 4. 工单管控（FR-ORD-4x）

## 5. 关键业务流程（主流程示意）

```
ERP 工单 ─► 接收/建单 ─► 下达(校验) ─► 排程发布 ─► 工位派工队列
   ─► 上岗 ─► 开工(防错校验) ─► [过程报数/参数采集/巡检] ─► 完工
   ─► 质量门检验 ─► 放行 ─► 下道工序 …… ─► 末道完工 ─► 产出批次入库
   ─► 工单完工 ─► 数据回传 ERP ─► 关闭
```
