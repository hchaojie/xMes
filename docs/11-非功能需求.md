# 11 非功能需求（NFR）

## 1. 性能与容量

| 编号 | 指标 | 目标 |
| --- | --- | --- |
| NFR-01 | 终端报工响应 | P95 < 1s（含防错校验） |
| NFR-02 | 看板/监控实时性 | 事件发生→大屏刷新 < 5s |
| NFR-03 | 排程甘特加载 | 200 工位 × 5000 作业首屏 < 3s，拖拽交互 60fps |
| NFR-04 | 追溯查询 | 5 层谱系全链路 < 3s |
| NFR-05 | 设备事件接入吞吐 | 设备事件接入 API（FR-INT-20）入库峰值 ≥ 2000 TPS（为物联网模块接入预留）；采集侧指标属物联网模块项目 |
| NFR-06 | 容量基线 | 500 工位、2000 设备、并发终端 500、年报工事件 5000 万、明细在线保留 24 个月 |

## 2. 可用性与可靠性

| 编号 | 要求 |
| --- | --- |
| NFR-11 | 系统可用性 ≥ 99.5%（生产时段）；网关/核心服务无单点 |
| NFR-12 | 报工链路降级：分析/看板故障不影响报工；事件分发组件（Redis Stream）不可用时报工仍同步落库（outbox），恢复后自动补发 |
| NFR-13 | 终端弱网容错：离线排队补传（FR-SHF-15） |
| NFR-14 | 数据备份：数据库每日全备 + binlog 增量，RPO ≤ 5min，RTO ≤ 2h |
| NFR-15 | 幂等：所有接口与事件消费幂等（业务键去重），补传/重试不产生重复数据 |

## 3. 安全

| 编号 | 要求 |
| --- | --- |
| NFR-21 | 认证授权沿用 pig OAuth2.1；工位终端使用受限 scope 客户端；API 客户端凭证授权 |
| NFR-22 | 数据权限按组织节点（工厂/车间/产线）过滤（01 §6），并支持 MIP 式"责任区域"（Responsibility Area）保护人员等敏感数据 |
| NFR-23 | 审计跟踪（对应 MIP 安全设计）：登录、关键单据状态变更、补录/冲销、防错放行、配置变更，以及对敏感对象与数据的访问全审计，留存 ≥ 3 年 |
| NFR-23a | 双人复核（四眼原则）：平台级"二次授权"能力——指定操作（让步接收、防错放行、结果记录修正等）须第二人刷卡/登录批准，可按操作类型配置启用 |
| NFR-23b | 数字签名：关键确认（首件放行、检验判定、SOP 阅读确认、交接班）以个人凭证（刷卡/密码/扫码）形成可追溯的电子签名记录 |
| NFR-24 | 传输加密 HTTPS/WSS；敏感字段（人员手机号等）存储加密；日志脱敏 |
| NFR-25 | 私有化部署内网隔离；设备网与办公网间仅开放采集服务端口 |

## 4. 可维护与可观测

| 编号 | 要求 |
| --- | --- |
| NFR-31 | 全链路日志 traceId（沿用 pig/SkyWalking 或 Micrometer Tracing）；慢接口、MQ 积压、采集断线告警 |
| NFR-32 | 配置中心化（Nacos），业务开关（防错项、校验级别、容差）运行时可调 |
| NFR-33 | 数据库变更脚本版本化（Flyway/Liquibase），支持平滑升级 |

## 5. 易用性与国际化

| 编号 | 要求 |
| --- | --- |
| NFR-41 | 终端 UI 遵守 FR-SHF-13 现场可用性规范；管理端关键流程 ≤ 3 次点击可达 |
| NFR-42 | i18n：前后端文案资源化，首期中文，预留英文包（vben 自带 i18n 机制） |
| NFR-43 | 单位/时区：时间统一 UTC 存储、按 Site 时区展示；数量精度按物料单位配置 |

## 6. 兼容性

| 编号 | 要求 |
| --- | --- |
| NFR-51 | 浏览器：Chrome/Edge 最近 2 个大版本；工位终端 WebView ≥ Chromium 90 |
| NFR-52 | 服务器：x86_64 与 ARM64（信创预留：达梦/人大金仓适配列为 P2 评估项） |
