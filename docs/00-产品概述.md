# 00 产品概述

## 1. 背景与定位

xMes 是一套面向离散制造（机加、装配、注塑、钣金、电子等行业）的车间制造执行系统（MES），定位于 ERP 计划层与车间设备控制层之间，承担 IEC 62264（ISA-95）Level 3 的制造运营管理（MOM）职能：

```
 L4  ERP（计划层）        —— 销售订单、主生产计划、物料需求、财务
─────────────────────────────────────────────────────────────
 L3  xMes（执行层）       —— 工单执行、排程、数据采集、质量、追溯、物流
─────────────────────────────────────────────────────────────
 L2/L1  SCADA/PLC/设备    —— 设备控制、传感器、计数器
```

产品功能体系参照 MPDV HYDRA X 的模块划分（订单管理、资源管理、物料管理、装配管理、质量管理、人员管理、信息管理七大类），并按国内离散制造车间的实际场景裁剪与本地化。

## 2. 产品目标

| 目标 | 说明 | 度量 |
| --- | --- | --- |
| 生产透明化 | 实时回答"什么订单、在哪台设备、由谁、生产到什么程度" | 现场数据上报延迟 < 5s |
| 缩短制造周期 | 通过精细排程、减少等待与寻料时间 | 订单平均周期时间下降 |
| 提升设备效率 | 自动采集设备状态，量化并改善 OEE | OEE 可按班次/设备/产品统计 |
| 质量内建与追溯 | 制程检验、SPC、防错；正反向批次追溯 | 追溯查询 < 3s 出全链路 |
| 无纸化车间 | 电子作业指导、电子检验单、电子报工 | 关键流程 100% 无纸化 |
| 数据驱动决策 | KPI 驾驶舱、异常升级、持续改善闭环 | 标准 KPI 体系（参照 VDMA 66412） |

## 4. 产品范围（模块地图）

参照 MPDV HYDRA X 的 mApp 分类，xMes 规划为 7 个业务域 + 1 个平台域：

```
┌─────────────────────────────────────────────────────────────────────┐
│                      信息与分析 (Information)                        │
│   驾驶舱 · 车间看板 · KPI/OEE · 升级管理(Escalation) · 措施管理 · 文档 │
├──────────────┬──────────────┬──────────────┬─────────────────────────┤
│  订单管理     │  资源管理     │  物料管理     │  质量管理               │
│  (Order)     │  (Resource)  │  (Material)  │  (Quality)              │
│  工单管理     │  设备管理     │  车间物流/WIP │  检验计划/执行(FEP)      │
│  作业排程(HLS)│  设备数采(MDE)│  批次/序列号  │  SPC                    │
│  生产报工(BDE)│  维护保养(MPM)│  追溯(TRT)   │  不合格品(NCM)           │
│  工单管控     │  工装模具(WRM)│  线边库       │  检具管理(PMV)           │
│              │  NC程序(DNC)  │  配送任务     │  投诉管理(REK)           │
│              │  能源(EMG)    │              │                         │
├──────────────┴──────┬───────┴──────────────┴─────────────────────────┤
│   人员管理 (HR)      │       装配与现场执行 (Assembly & Shopfloor)     │
│   工时采集(PZE)      │       工位终端(AIP) · 电子作业指导 · 防错        │
│   排班(PEP)·资质(PQ) │       安灯(Andon) · 交接班                      │
├─────────────────────┴───────────────────────────────────────────────┤
│                平台域：主数据 · 车间建模 · 集成(ERP/设备) · 权限        │
│                （基于 pig：用户/角色/菜单/租户/网关/认证）              │
└─────────────────────────────────────────────────────────────────────┘
```

> 括号内缩写沿用 MPDV HYDRA 经典模块命名，便于与参照产品对照：
> BDE=生产数据采集，MDE=机器数据采集，HLS=电子排程看板，MPM=维护管理，
> WRM=工装模具管理，PMV=检具管理，FEP=制程检验，REK=投诉管理，
> TRT=批次追溯，PZE=人员工时，PEP=人员排班，DNC=NC程序传输，EMG=能源管理。

### 4.1 范围内（In Scope）

- 车间静态建模（工厂/车间/产线/工位/设备/班次/日历）与制造主数据（物料/BOM/工艺路线/工序）
- 工单全生命周期管理与精细化作业排程
- 现场执行：报工、数采、工位终端、作业指导、安灯
- 设备：状态采集、OEE、维护保养、备件
- 物料：车间内物流、线边库、批次/序列号、正反向追溯
- 质量：进料/制程/终检、SPC、不合格品闭环、检具周检
- 人员：工时、排班、资质匹配
- 分析：KPI、驾驶舱、看板、异常升级
- 集成：ERP 接口、设备联网、开放 API

### 4.2 范围外（Out of Scope，首版不做）

- 财务成本核算（仅提供工时/物耗数据给 ERP）
- 完整 WMS 仓库管理（仅做车间/线边库，与仓库系统对接）
- 完整 APS 有限产能自动优化排程（首版提供人工交互式排程 + 规则排序，自动优化算法列为 P2）
- 薪资计算（仅输出考勤工时与计件数据）
- PLM/CAPP（工艺数据可从外部导入）

## 5. 设计原则

1. **对象模型先行**：所有功能建立在统一的车间对象模型（见 [02 车间对象模型](02-车间对象模型.md)）之上，参照 ISA-95 的设备层级模型与 MPDV MIP 平台"语义信息模型"思想——各模块共享同一套对象，而非各自建表。
2. **平台 + 应用**：平台域（建模、主数据、集成、权限）与业务 mApp 解耦，业务模块可按需启用（对应 HYDRA X 按 mApp 订购的产品形态）。
3. **事件驱动**：现场产生的是不可变的事件流（报工、状态变更、检验记录），统计视图由事件推导，保证可审计、可重算。
4. **离散制造优先**：首版面向离散行业；流程行业（配方、罐区）的扩展预留对象模型挂点，不在首版实现。
5. **渐进式交付**：按 [12 实施路线图](12-实施路线图.md) 分期上线，每期可独立产生业务价值。

## 6. 术语表

| 术语 | 英文/缩写 | 定义 |
| --- | --- | --- |
| 制造执行系统 | MES | 面向车间执行层的管理系统 |
| 制造运营管理 | MOM | ISA-95 Level 3 各类运营活动的总称 |
| 工作中心 | Work Center | 产能与排程的基本单位，可对应产线、班组或设备组 |
| 工位 / 机台 | Workplace / Work Unit | 执行工序的最小物理单元，ISA-95 中的 Work Unit |
| 工单 / 生产订单 | Production Order | 下达到车间的生产任务，通常源自 ERP |
| 工序 | Operation | 工艺路线中的一个加工步骤 |
| 作业 | Operation Task | 工单的某道工序在车间的执行实例（排程与报工的对象） |
| 报工 | Booking / Confirmation | 现场对作业的开工/完工/数量/工时申报 |
| 在制品 | WIP | Work in Process，车间内未完工的物料 |
| 批次 | Lot / Batch | 同一生产条件下产出、质量特性一致的物料集合 |
| 设备综合效率 | OEE | 时间开动率 × 性能开动率 × 合格品率 |
| 统计过程控制 | SPC | 基于控制图的过程质量监控方法 |
| 不合格品 | NC / NCM | Nonconformance，检验不通过的产品及其处置流程 |
| 安灯 | Andon | 现场异常呼叫与响应机制 |
| 首件检验 | FAI | 批量生产前对首件产品的检验确认 |
| 电子排程看板 | Leitstand / 甘特图 | 以甘特图交互方式进行作业排程的工具 |
| 数控程序传输 | DNC | 向数控设备分发/回传加工程序 |
