# 12 实施路线图

按"每期独立产生业务价值"切分为四期。各期范围引用功能编号（FR-xxx），优先级口径与各需求文档一致。

## Phase 1 —— 基础执行闭环（P0 核心，建议 3~4 个月）

**目标：跑通"工单下达 → 排程 → 终端报工 → 进度可视 → 回传 ERP"主闭环。**

| 域 | 范围 |
| --- | --- |
| 平台 | pig 骨架落地、组织/权限/数据权限、车间建模（Site~Workplace、库位）、班次日历、编码规则 |
| 主数据 | 物料、BOM、工艺路线/工序、工艺参数模板（FR 见 02 模型） |
| 订单 | FR-ORD-11~14、16、18（含工作计划维护与生成工单）；排程 FR-ORD-21/22/24/26/27（自研甘特）；报工 FR-ORD-31~34、36（含结果记录与签核）；管控 FR-ORD-41 |
| 资源 | FR-RES-11~14、21、22、25、2A4（设备建模+状态采集+监控墙+OEE/NEE） |
| 物料 | FR-MAT-11~14、21、23、24、31、35；追溯 FR-MAT-41/42 |
| 质量 | FR-QUA-11~14、21、22、24、25、41、42（检验闭环+不合格品闭环） |
| 人员 | FR-HRM-11、12、21、22、31 |
| 现场 | 工位终端框架 FR-SHF-11~14、16；SOP 展示 FR-SHF-21；防错框架 FR-SHF-31、34；安灯 FR-SHF-41、42 |
| 分析 | FR-INF-11~13、21、22（OEE+驾驶舱+大屏） |
| 集成 | FR-INT-11~16、21、22、26（ERP 闭环+OPC UA/MQTT+扫码打印） |

**里程碑**：M1 平台+建模+主数据（第 6 周）；M2 工单+报工+终端（第 12 周）；M3 质量+追溯+看板+ERP 联调（第 16 周）。

## Phase 2 —— 精益与质量深化（P1 为主，约 3 个月）

- 排程深化：表格化顺序排程+换型/物料需求清单、规则排序、作业拆分与锁定、负荷分析、换型矩阵、生产变体（FR-ORD-23/25/28/2A/2B）
- 报工深化：多人多机与个人工时（POE）、外协、异常待办、工时票（FR-ORD-35/37/38/39）；管控分析（FR-ORD-43/44）
- 维护保养与点检（FR-RES-31~33、35、36）、模具管理（FR-RES-41~43）、计数自动报工与过程参数（FR-RES-23/24/27）、设备状态分析包（FR-RES-2A1~2A3、2A5）
- 物流深化：叫料/配送、移转、包装层级、批次拆合、代用（FR-MAT-15/16/22/25/32/33/34/43）
- 质量深化：SPC（FR-QUA-31~33）、检具（FR-QUA-51/52）、客诉登记（FR-QUA-61）
- 人员深化：排班、资质拦截、工时汇总（FR-HRM-23/24/32~34/41/42）
- 现场深化：步骤引导、参数面板、顺序防错、交接班、离线排队（FR-SHF-15/22/23/24/32/51）
- 信息深化：升级管理、措施管理、文档管理、报表订阅、多维下钻、车间平面布局编辑器（FR-INF-14/22a/24/31/32/41/51/52）
- 人员深化补充：工牌/替代卡管理（FR-HRM-11a）
- 集成深化：OpenAPI、消息推送、边缘缓存、接入健康度（FR-INT-23/25/31/33）
- 时序库引入（架构 D3 决策落地）

## Phase 3 —— 智能与扩展（P2，按需启动）

- 排程智能化：辅助排程、排程模拟、APS 自动优化（FR-ORD-2C/2D/29）、工单合并/成本归集（FR-ORD-15 合并、45）
- DNC 程序下发回传（FR-RES-52/53）、备件（FR-RES-34）、能源管理（FR-RES-6x）
- 8D 完整流程与供应商质量（FR-QUA-62/63）、量具直连（FR-QUA-23）、参数 SPC（FR-QUA-34）
- 技能矩阵（FR-HRM-43）、外设防错联动（FR-SHF-33）、班会看板（FR-SHF-52）
- 看板配置化（FR-INF-23）、效果验证闭环（FR-INF-42）、Webhook（FR-INT-32）、追溯性能专项（图查询/ES）
- 信创数据库适配评估（NFR-52）
